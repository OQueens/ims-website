# Sol adversarial audit — LIVE rate simulator (2026-07-10)

Scope: production src/lib/rate-engine/*.ts and src/lib/hub/*.ts, plus the live hub caller needed to prove reachability. Verdict standard: a concrete input must reach a wrong or unjustifiably confident displayed dollar figure. No tests were run.

## A. Independent verdicts on C1–C17

- **C1 — REAL; High.** mapSpecialty returns source:'inferred' for any alias/key substring without checking unmatched specialty-bearing tokens (rateCalculator.ts:82-92), and the hub treats every non-default specialty as quotable (sim-adapter.ts:185-198); e.g. PDF Specialty: Radiation Oncology reaches medical-oncology dollars, although static data usually limits displayed confidence to Medium (sim-adapter.ts:305-310), not always High as claimed.
- **C2 — REFUTED; no quote severity.** The internal-medicine fallback exists (rateCalculator.ts:77-94), but both live PDF and freetext adapters reject specialty.source === 'default' before quoting (sim-adapter.ts:185-188,203-207); the cited CNM/spine/transplant/addiction inputs produce manual-escalation UI, not an internist quote.
- **C3 — REAL; Medium.** Manual controls expose no facility/call/holiday fields (sim-adapter.ts:38-46) and silently construct community/no-call/no-holiday factors (sim-adapter.ts:127-145), so a recruiter manually modeling a known CAH assignment cannot supply its 1.22 factor (multipliers.ts:23-35) and gets the neutral quote.
- **C4 — REAL; High.** Bare oncology maps to medical oncology (specialties.ts:288) and all aliases run before canonical-key substrings (rateCalculator.ts:82-92), so Gyn Oncology reaches $350–500 medical oncology instead of the existing $260–370 gynecologic-oncology cell (specialties.ts:105,117).
- **C5 — REAL; High.** endo is absent from the word-boundary alias set (rateCalculator.ts:74-87), so Endovascular Surgery matches it before any specialty key and is quoted from endocrinology (specialties.ts:85,98,102,283).
- **C6 — REAL; Medium.** Parsed factors freeze the then-current spec.p70 (rateCalculator.ts:877-902); after the async overlay mutates spec.p70 (marketRates.ts:519-527), the live re-render reuses pendingFactors (hub-client.ts:248-253,522-535) and calculateRate prefers that stale nonzero base (rateCalculator.ts:656-680).
- **C7 — REAL; Medium.** Structured Holiday Coverage: Yes drives only hasHoliday (rateCalculator.ts:360-394) and therefore 1.10x, while notes containing bare holiday drive shift='holiday' (rateCalculator.ts:213-235) and therefore 1.35x; the parse channel, not a proved burden difference, selects the premium (rateCalculator.ts:670-675).
- **C8 — REAL; Medium.** Any 24-hour, 24hr, or 24/7 text is unconditionally classified as night before weekend/day evidence (rateCalculator.ts:213-235), applying the 1.20 night factor to the entire hourly assignment (multipliers.ts:13-19; rateCalculator.ts:670-680).
- **C9 — REAL; High.** Promotion checks positivity/corroboration but no upper plausibility bound (marketRates.ts:493-519), then makes the anchor p70 and derives a larger ceiling from it (marketRates.ts:527-545); a fresh two-family $900 posterior can therefore become a High-confidence $900 base rather than being bounded by the researched prior.
- **C10 — REAL; Low.** The displayed p90 is linear band geometry (sim-adapter.ts:354-372) but is labeled a dollar-valued Premium tier (sim-render.ts:50-69), presenting an unobserved interpolation as an actionable market segment even though it does not alter the hero.
- **C11 — REAL; High.** The promotion gate counts every normalized nonblank string as an independent family (marketRates.ts:499-517), including unknown/unattributed; one real family plus that sentinel can unlock an arbitrary High-confidence anchor, so this is material anti-manipulation exposure, not Low.
- **C12 — REAL; Medium.** The hourly engine deliberately fixes ruralMult=1 (rateCalculator.ts:656-675) and the manual path cannot select CAH/rural-trauma (sim-adapter.ts:127-145); consequently a known rural/CAH manual placement is priced from only a state aggregate, while 1.22/1.30 facility factors exist but are unreachable (multipliers.ts:23-35).
- **C13 — REFUTED; no quote severity.** Hyphens are stripped while / is retained (parser.ts:272-277), so ob-gyn can fail specialty matching, but buildParsedFromFreetext then returns null (parser.ts:279-305,356-360); this is an availability defect, not a wrong or over-confident quote.
- **C14 — REAL; Medium.** Call burden is reduced to a boolean (rateCalculator.ts:323-337) and every true value receives the same uncited 1.10 factor (rateCalculator.ts:670-675), so q2 in-house call and 1:4 shared beeper call produce the same dollar adjustment without a burden-confidence downgrade.
- **C15 — REFUTED for current production; latent hazard only.** The reader really does choose national then the first state cell without receiving the quote state (marketRates.ts:284-313), but the supplied production evidence says v2 is national-only; absent a current state-only cell, no concrete live user input triggers the claimed cross-state misquote.
- **C16 — REAL; Medium.** Bare icu is a non-boundary alias to adult critical care (specialties.ts:263; rateCalculator.ts:74-88), and refinement only converts a starting pediatrics key (rateCalculator.ts:105-125), so PDF Pediatric ICU reaches adult $190–300 instead of pediatric-critical-care $180–260.
- **C17 — REFUTED; no defect severity.** The engine intentionally bounds extrapolated multiplier math to spec.max (rateCalculator.ts:693-707) and the live UI explicitly says the quote was bounded (sim-render.ts:78-81); two inputs sharing that evidence ceiling does not prove the displayed dollars wrong or over-confident.

## B. New issues, ranked

1. **High — A parsed hourly bill-rate cap can be exceeded by the displayed bill.** rateCalculator.ts:683-691 converts every hourly cap to pay with a fixed 0.80 factor, but sim-adapter.ts:298-300,354-361 recomputes bill from the user-selected margin. Concrete input: anesthesiology PDF with Rate Cap: $250/hr, default 22% margin → pay is capped to $200, then the hub displays roundUp5(200/0.78) = $260/hr, above the contractual $250 cap.

2. **High — Alias-first routing misprices known canonical specialties when ordinary suffixes/variants prevent exact equality.** rateCalculator.ts:77-92 scans every alias before canonical-key substrings. Concrete inputs: PDF Reproductive Endocrinology and Infertility hits endo and quotes adult endocrinology $200–275 instead of the existing reproductive-endocrinology $230–340 cell; Urogynecology Physician hits gynecology and quotes OB/GYN $160–275 instead of urogynecology $200–290; Pediatric Anesthesiologist hits anesthesiologist and quotes general rather than pediatric anesthesia (specialties.ts:58,62,102,114,116,118,248,283,291).

3. **High — Facility inference applies negated and lower-priority facility words as real price factors.** inferFacilityTypeEnhanced has no negation handling and returns on first category match (rateCalculator.ts:238-298). Concrete input: community-hospital PDF notes No telehealth; provider must be on site still select telehealth and multiply pay by 0.75 (multipliers.ts:35); university-affiliated critical access hospital selects academic 0.85 before CAH 1.22 (rateCalculator.ts:248-261), a 30% relative underquote before ceilings.

4. **High — The market promotion gate has no researched-prior lower bound.** It accepts any finite weighted_mean > 0 (marketRates.ts:493-519) and widens spec.min downward to the anchor (marketRates.ts:527-528). Concrete market input: fresh anesthesiology actual_paid_locum bucket with weighted_mean:50, n_distinct:4, and two family strings promotes a High-confidence $50 base despite the curated $300–400 range; this is the unguarded low-side mirror of C9.

5. **High — Bucket confidence is validated but not enforced at promotion.** The reader accepts single_source as renderable (marketRates.ts:200-234), while promotion ignores b.confidence and trusts the separately supplied family array (marketRates.ts:483-517). Concrete market input: a fresh bucket marked confidence:'single_source', n_distinct:4, source_families:['amn','unattributed'], weighted_mean:600 passes and is relabeled High at marketRates.ts:546-551.

6. **Medium — Far-future timestamps remain fresh indefinitely.** isFreshBucket only tests now - lastUpdated <= 7 days and has no future-skew bound (marketRates.ts:155-164). Concrete market input: an otherwise promotable bucket stamped ten years ahead survives every freshness check for roughly ten years and can keep an obsolete/wrong anchor live.

7. **Medium — An ambiguous cap is silently applied as hourly and its warning never reaches the hub quote.** extractBillRateCap marks a unitless cap unit:'unknown', hasWarning:true (rateCalculator.ts:503-541), yet hourly math applies unknown as a cap (rateCalculator.ts:683-691); SimQuote carries no rate-cap warning (sim-adapter.ts:220-252) and the renderer only explains numerical clamps (sim-render.ts:78-88). Concrete input: hourly PDF note NTE $250 with no unit is silently treated as $250/hour rather than escalated.

8. **Medium — Multi-state assignments are quoted from the first facility and can display High identification confidence.** inferStateEnhanced returns immediately on the first facility state (rateCalculator.ts:137-140), and scoreConfidence treats that as known geography (rateCalculator.ts:621-637). Concrete input: one PDF listing facilities in Denver, CO and Cheyenne, WY receives only the CO scalar and state label even when the placement being priced is the WY site.

9. **Medium — Reversed/invalid assignment dates create an emergency premium instead of rejection.** inferDuration computes signed days and classifies every value <= 7 as emergency (rateCalculator.ts:301-319). Concrete input: a PDF typo Start Date: 08/15/2026, End Date: 08/14/2026 yields the 1.20 emergency multiplier (multipliers.ts:38-43) and a higher quote rather than an invalid-date/manual-review state.
