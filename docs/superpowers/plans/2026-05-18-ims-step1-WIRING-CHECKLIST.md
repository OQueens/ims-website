## Zach's wiring actions (required for `/jobs` to show real jobs)

`wrangler` is not authenticated in the build environment, so these cannot be
automated — do them in the Cloudflare dashboard / LocumSmart UI.

### 1. Cloudflare Pages env vars — project `ims-website`

Add these to the **`ims-website`** Pages project, on **BOTH** the
**Production** and **Preview** environments (a branch deploy of
`feat/ims-temp-site` inherits Preview):

| Variable | Type | Value |
|---|---|---|
| `SUPABASE_URL` | Plaintext | the IMS Supabase project URL (`https://<ref>.supabase.co`) |
| `SUPABASE_SERVICE_ROLE_KEY` | **Secret (encrypted)** | the Supabase **service-role** key (server-only; never the anon key, never client-exposed) |
| `LOCUMSMART_WEBHOOK_SECRET` | **Secret (encrypted)** | the Webhook Key auto-generated by LocumSmart when you create the "Website" subscription in step 2 |

Notes:
- `SUPABASE_SERVICE_ROLE_KEY` and `LOCUMSMART_WEBHOOK_SECRET` MUST be the
  encrypted **Secret** type, not plaintext.
- These coexist with the already-provisioned Resend vars (`RESEND_API_KEY`,
  `RESEND_FROM_EMAIL`, `RECRUITING_TO_ADDRESS`) — do not disturb those.
- Until all three are set, `/jobs` correctly shows the dignified empty state
  (the `unconfigured` path is **silent** — no log noise, no error header).

### 2. LocumSmart "Website" subscription — NO specialty filter

- Create a **dedicated** LocumSmart webhook subscription named
  **"Website"** with **NO specialty filter — every specialty** (this is
  distinct from the existing specialty-FILTERED LS→Slack subscription; the
  website needs the full, unfiltered feed).
- Point it at: **`https://innovativemedicalstaffing.com/api/locumsmart-webhook`**
- Copy the **Webhook Key** LocumSmart generates for that subscription into the
  Cloudflare Pages env var **`LOCUMSMART_WEBHOOK_SECRET`** (step 1) — exact
  match; the receiver does a constant-time compare on it.
- (Housekeeping, if still outstanding from earlier sessions: delete any stale
  discovery/test LS subscription and treat any previously-exposed Webhook Key
  as burned — rotate by regenerating the "Website" subscription key.)

### 3. First-event end-to-end verification (the real proof — see DoD caveat)

After steps 1–2, the first real LS assignment event will exercise the
**bundled data path** for the first time (the local gate runs without env so
it never executes the Supabase read). Verify once wired:
- LS event → `200 OK` from `/api/locumsmart-webhook`; a row appears in
  `ims_jobs`; `/jobs` then renders that job as a public-only card with **zero
  `<script>`** and the Belleval design intact.
- A deliberately wrong key → `401`; malformed body → `400`; missing env →
  `500` (LS retries 5xx, so a transient deploy gap self-heals).
- If Supabase is reachable but the read fails/stalls, `/jobs` still serves the
  dignified shell **and** sets the `x-jobs-data: degraded` response header +
  a distinct `DATA READ DEGRADED` log (so an outage is not mistaken for an
  empty board). A healthy-but-empty board sets **no** such header.
